Overview

Designed, implemented, and verified a dual-core matrix multiplying accelerator targeting the attention mechanism in transformer networks — taken from RTL all the way through place-and-route using industry-standard ASIC tools.

The project followed six steps: single-core RTL and synthesis, output normalization, hierarchical core synthesis with SRAM macros, dual-core integration with asynchronous clock-domain crossing, performance and power optimization, and sparsity-aware computation.

Key Contributions
  • Fixed output port mux, parallel factor, async reset, and latch inference issues in the starter RTL
  • Synthesized with Cadence Genus targeting 1 GHz / TSMC 65 nm standard cell library
  • Place-and-route with Cadence Innovus; verified with gate-level simulation + back-annotated SDF timing
  • Integrated two SRAM hard macros (128-bit and 160-bit) as fixed IP blocks with 3 µm routing halos
  • Built dual-core architecture with asynchronous FIFO for clock-domain crossing between cores
  • Achieved 0 DRC errors on the final dual-core design; all 64 normalized outputs matched expected values
Design Steps
STEP 01
Single-Core RTL, Synthesis & PnR
Corrected starter RTL: fixed output mux, reduced parallel factor from 16 to 8, added async reset, and eliminated latch inference. Synthesized with Cadence Genus at 1 GHz targeting TSMC 65 nm. Placed and routed with Cadence Innovus. Post-PnR: WNS −1.216 ns, 169.7 mW, 98.8% utilization. Gate-level simulation with SDF verified all Q·K products at 1680 ns.
STEP 02
Output Normalization (SFP)
Implemented the sfp_row normalization module connected to the partial-sum memory output. Two-phase operation: accumulation phase computes per-row divisors (sum of absolute column values), stored in an internal FIFO; division phase normalizes each element and writes back via a pipeline mux. Behavioral simulation verified correct normalized values for all 8 rows × 8 columns.
STEP 03
Hierarchical Single-Core Synthesis
Instantiated two SRAM macros as fixed hard IP blocks: sram_128b_w16 (128-bit, 16 entries) for Q/K memory and sram_160b_w16 (160-bit, 16 entries) for psum memory. Each macro placed with a 3 µm routing halo and its own VDD/VSS block ring on M1/M2. Post-PnR: 231.0 mW, 960k / 1,000k µm² area. 0 DRC/LVS geometry violations.
STEP 04
Dual-Core Integration
Instantiated two single-core blocks in parallel: Core 0 processes K₀–K₇ and Core 1 processes K₈–K₁₅, each with its own 8-wide datapath. Cores exchange partial sums across clock domains via an asynchronous FIFO. Post-PnR: 382.3 mW, 1,262k / 3,744k µm², 99.0% utilization. Gate-level simulation confirmed 64 normalized outputs matched expected values in 150 ns.
STEP 05
Performance & Power Optimization
Applied four optimizations: pipelined MAC datapath (registered products, balanced reduction tree, registered outputs); multi-stage SFP pipeline replacing single-cycle division; activity gating on SRAMs to reduce dynamic power; and memory double buffering (ping-pong Q-memory banks). Result: WNS improved to −2.452 ns, power reduced to 370.08 mW, area reduced to 2,528,800 µm².
STEP 06
Sparsity-Aware Attention
Added row-level sparsity detection in the sfp_row module. Each row's sparsity score Sᵢ is computed; rows below the 30th-percentile threshold are marked sparse and skip normalization entirely — outputs are zeroed. Sorted Sᵢ values to select the threshold data-adaptively. With 30% sparsity, roughly 30% of rows complete in one cycle. Final: 378.21 mW, 1,000,000 µm² area, 1.07% power reduction over baseline.
